# Техническое описание приложения RecipeFinder

## Обзор
RecipeFinder — это Android-приложение для поиска кулинарных рецептов на основе имеющихся у пользователя ингредиентов. Приложение позволяет выбирать продукты, которые есть под рукой, и предлагает наиболее подходящие рецепты, сортируя их по количеству совпадений.

## Архитектура
Приложение спроектировано с использованием принципов **Clean Architecture** и паттерна **MVVM (Model-View-ViewModel)**. Код разделен на слои:

1.  **Data Layer (Слой данных):**
    *   `JsonDataSource`: Загружает данные из локальных JSON-файлов в папке `assets`.
    *   `Repositories`: Обеспечивают доступ к данным об ингредиентах и рецептах, скрывая детали реализации (загрузку из JSON).
2.  **Domain Layer (Слой бизнес-логики):**
    *   `Models`: Сущности `Recipe` (Рецепт) и `Ingredient` (Ингредиент).
    *   `UseCases`: Отдельные классы для бизнес-задач:
        *   `SearchIngredientsUseCase`: Поиск ингредиентов по строке.
        *   `GetRecommendedRecipesUseCase`: Алгоритм подбора рецептов.
        *   `GetIngredientByIdUseCase`: Получение информации о конкретном ингредиенте.
3.  **Presentation Layer (Слой представления):**
    *   `MainViewModel`: Хранит состояние экрана (поисковый запрос, корзина, список рецептов) и обрабатывает действия пользователя.
    *   `UI (Jetpack Compose)`: Экраны приложения (`MainScreen`, `RecipesScreen`, `RecipeDetailScreen`).
4.  **Navigation:** Используется `Jetpack Navigation Compose` для перемещения между экранами.

## Функционал
*   **Поиск ингредиентов:** Динамический поиск по базе продуктов.
*   **Корзина продуктов:** Возможность добавлять и удалять выбранные ингредиенты.
*   **Умный подбор:** Алгоритм рассчитывает "счёт" (score) для каждого рецепта на основе процента имеющихся ингредиентов и сортирует результаты.
*   **Детали рецепта:** Отображение времени приготовления, сложности, списка шагов и списка недостающих продуктов.

## Технологический стек
*   **Язык:** Kotlin
*   **UI:** Jetpack Compose
*   **Архитектура:** MVVM + Clean Architecture
*   **Навигация:** Navigation Compose
*   **Данные:** JSON (Assets)
