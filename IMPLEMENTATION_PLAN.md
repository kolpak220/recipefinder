# План внедрения новых функций

Этот документ описывает шаги по внедрению системы маппинга ингредиентов и интеграции ИИ для оценки релевантности рецептов.

## 1. Система маппинга ингредиентов (Синонимы и замены)

Цель: позволить системе учитывать взаимозаменяемые продукты (например, "Сыр" -> "Моцарелла").

### Шаги реализации:
1.  **Data Layer:**
    *   Создать файл `assets/ingredient_mappings.json`, где ключом будет ID основного ингредиента, а значением — список ID его вариаций/замен.
    *   Добавить метод в `JsonDataSource` для загрузки этих связей.
    *   Обновить `IngredientRepository`, добавлена поддержка метода `getRelatedIngredientIds(id: Int): List<Int>`.
2.  **Domain Layer:**
    *   Создать `ResolveIngredientSubstitutionsUseCase`.
    *   **Рефакторинг `GetRecommendedRecipesUseCase`**:
        *   При расчете совпадений для каждого ингредиента рецепта проверять не только прямое наличие в корзине, но и наличие его "аналогов" через маппинг.
3.  **Логика работы:**
    *   Если в рецепте указан "Сыр" (ID 10), а у пользователя в корзине "Моцарелла" (ID 15), и они связаны в маппинге, рецепт получит балл за совпадение.

---

## 2. ИИ для оценки совпадения (Топ-10 рецептов)

Цель: использовать LLM (например, Google Gemini) для качественного анализа того, насколько хорошо имеющиеся продукты подходят для конкретного блюда, вместо простого математического процента.

### Шаги реализации:
1.  **Подготовка:**
    *   Добавить зависимость `com.google.ai.client.generativeai` в `build.gradle`.
    *   Получить API Key и добавить его в `local.properties` (не коммитить!).
2.  **Data/Domain Layer:**
    *   Создать `AiRepository` для взаимодействия с API.
    *   Создать `EvaluateRecipeCompatibilityUseCase`.
    *   **Входные данные**: Список названий ингредиентов пользователя + Данные рецепта (название, все ингредиенты, шаги).
    *   **Промпт**: "У меня есть [список]. Насколько хорошо они подходят для [Рецепт]? Оцени совместимость от 1 до 10 и дай краткий совет по замене недостающего."
3.  **Presentation Layer (`MainViewModel`):**
    *   После получения результатов от `GetRecommendedRecipesUseCase`, брать первые 10 позиций.
    *   Запускать корутину для асинхронного опроса ИИ для этих 10 рецептов.
    *   Добавить поле `aiEvaluation: String?` в модель отображения рецепта.
4.  **UI Layer:**
    *   В `RecipeCard` добавить блок "Мнение ИИ", который отображается, когда ответ получен.
    *   Добавить индикатор загрузки (Shimmer) для текста от ИИ.

## Приоритеты
1.  **Этап 1**: Маппинг ингредиентов. Это значительно улучшит точность поиска без затрат на API и задержек сети.
2.  **Этап 2**: Интеграция ИИ. Добавит "вау-эффект" и поможет пользователю понять, как лучше адаптировать рецепт под свои продукты.
